# 未雨课表测试交付清单

这份文档用于你在把项目交给测试部门时，快速确认还需要准备哪些材料。

## 测试部门建议收到的内容

### 1. 测试版本

- 小程序体验版二维码，或明确可访问的测试环境
- 本次提测版本号 / 提测日期
- 对应代码提交号

### 2. 基础信息

- 产品名称：未雨课表
- 产品形态：微信小程序
- 项目定位：多人自学学习管理工具
- 当前测试范围：课程、课表、打卡、学分、奖励、统计、个人中心

### 3. 测试账号与访问方式

- 至少 2 个测试微信账号
- 哪些账号已经加入体验版白名单
- 是否需要区分不同用户的数据隔离测试

### 4. 测试前置数据

- 建议预置至少 3 门共享课程
- 建议预置至少 1 个已完成排期的用户
- 建议预置至少 1 个已有打卡记录的用户
- 建议预置至少 1 套学分规则和 1 个奖励目标

### 5. 功能说明材料

- [02_使用说明.md](./02_使用说明.md)
- [03_测试说明.md](./03_测试说明.md)
- [04_测试用例.md](./04_测试用例.md)
- [05_缺陷提报模板.md](./05_缺陷提报模板.md)

### 6. 已知限制或不测范围

- 暂不覆盖 AI 能力
- 暂不覆盖真实文件上传
- 暂不覆盖海报生成
- 暂不覆盖复杂提醒通知
- 暂不覆盖大规模并发场景

### 7. 测试反馈要求

建议测试部门反馈时至少包含：

- 问题标题
- 问题等级
- 复现步骤
- 实际结果
- 期望结果
- 截图 / 录屏
- 设备信息
- 微信版本

## 你还需要补给测试部门的东西

如果你准备正式发给测试部门，除了这套文档，我建议你再补这 5 项：

1. 体验版二维码
2. 本次提测版本号
3. 测试账号名单
4. 已知问题清单
5. 提测范围与不测范围

## 建议提测附言

可以直接附上这段说明：

```text
本次提测范围为：课程录入与查看、个人课表编排、学习打卡、学习记录、学分规则、奖励目标、统计中心、个人资料与主题切换。

请重点关注：
1. 用户数据隔离是否正确
2. 打卡流程与状态切换是否正确
3. 学分与奖励进度是否同步更新
4. 课程排期与课表展示是否一致
5. 首页、统计页、奖励页的数据是否一致
```
