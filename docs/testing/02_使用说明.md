# 未雨课表使用说明

## 一、产品用途

未雨课表是一个面向长期学习场景的微信小程序，核心用途是把课程管理、固定学习排期、每日打卡、学习记录、学分激励和奖励进度统一管理。

适合场景：

- 自学型学习计划
- 在职人员下班后长期学习
- 多个成员分别管理自己的学习计划

## 二、当前主要功能

### 1. 首页

- 查看今日学习安排
- 查看本周进度
- 查看主奖励进度
- 查看最近学习记录

### 2. 课程

- 查看共享课程库
- 新增课程
- 查看课程详情
- 为课程编排学习排期
- 查看个人课表

### 3. 打卡

- 查看今日待学习课程
- 进行上课打卡
- 完成下课打卡
- 填写学习记录
- 将课程标记为请假 / 休息 / 旷课

### 4. 我的

- 维护个人资料
- 维护学分规则
- 维护奖励目标
- 查看统计中心
- 切换主题配色

## 三、标准使用流程

建议按下面流程使用：

1. 首次进入小程序
2. 在“课程”页录入共享课程
3. 在课程详情或课表页为自己编排学习时间
4. 到了学习时间，在“打卡”页开始上课打卡
5. 学习完成后，执行下课打卡并填写学习记录
6. 回到首页 / 统计页查看学分、出勤和奖励进度变化

## 四、页面说明

### 首页

作用：

- 聚合展示当天与本周信息
- 展示当前学分与连续学习天数
- 展示主奖励进度

### 课程页

作用：

- 管理共享课程库
- 查看个人课表
- 快速跳转创建或编辑排期

### 打卡页

作用：

- 执行当天课程打卡
- 填写学习记录
- 维护打卡状态

### 个人中心

作用：

- 管理用户资料
- 管理个人规则
- 查看统计数据

## 五、关键业务规则

### 1. 用户数据隔离

- `courses` 为共享课程库
- 个人课表、打卡记录、学分规则、奖励目标按用户隔离

### 2. 打卡规则

- 课程开始前可选择：请假 / 休息 / 旷课
- 课程开始后进入“进行中”
- 完成下课打卡后进入“已完成”
- 已完成后可补充或修改学习记录

### 3. 学分规则

- 学分跟人走，不跟课程走
- 学分规则由用户自己配置
- 完成课程、深度学习、旷课等会影响学分

### 4. 奖励规则

- 每个用户维护自己的主奖励目标
- 奖励进度基于累计学分推进

## 六、测试使用建议

如果测试人员首次接触项目，建议按下面顺序体验：

1. 录入课程
2. 编排课表
3. 执行打卡
4. 修改学习规则
5. 查看奖励进度
6. 查看统计中心
7. 切换账号验证数据隔离
