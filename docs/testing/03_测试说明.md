# 未雨课表测试说明

## 一、测试目标

本次测试目标是验证未雨课表当前版本的核心学习闭环是否可用、稳定、一致：

```text
课程录入 -> 课表编排 -> 打卡 -> 学分变化 -> 奖励进度 -> 统计反馈
```

## 二、测试范围

### 本次测试范围

- 微信登录与用户初始化
- 首页数据展示
- 课程录入、编辑、详情查看
- 个人课表编排与编辑
- 打卡状态流转
- 学习记录保存
- 学分规则配置
- 奖励目标配置
- 统计中心
- 主题切换
- 多用户数据隔离

### 本次不在重点范围

- AI 相关能力
- 语音整理
- 海报生成
- 真正的文件上传
- 复杂消息提醒

## 三、测试方式建议

建议采用以下组合方式：

### 1. 功能测试

验证每个页面和业务功能是否按预期可用。

### 2. 业务流测试

验证跨页面链路是否闭环，例如：

- 录入课程后能否顺利进入排期
- 完成打卡后首页、奖励、统计是否同步变化

### 3. 异常测试

验证空数据、重复操作、非法输入、状态边界是否正常处理。

### 4. 隔离性测试

验证不同微信用户之间的数据不会串。

## 四、推荐测试环境

### 设备建议

- iPhone 1 台
- Android 1 台

### 客户端建议

- 微信最新正式版
- 微信开发者工具体验环境

### 账号建议

- 测试账号 A：主流程测试
- 测试账号 B：多用户隔离测试

## 五、重点关注项

测试时请重点关注以下问题：

### 1. 数据一致性

- 首页、统计页、奖励页展示的学分是否一致
- 打卡后统计是否实时同步
- 排期与课表展示是否一致

### 2. 状态流转正确性

- 待开始 -> 进行中 -> 已完成 是否顺畅
- 待开始状态下的请假 / 休息 / 旷课按钮是否符合预期
- 已完成状态是否还能正确保存学习记录

### 3. 用户隔离

- 用户 A 新建的排期、打卡、规则、奖励，用户 B 不应看到
- 共享课程库应允许多个用户查看

### 4. 边界输入

- 空课程名
- 非法时间范围
- 学分目标为 0
- 请假不填原因
- 休息次数超月上限

## 六、提测建议

如果测试部门时间有限，建议优先执行以下回归链路：

1. 首次进入小程序
2. 新增课程
3. 新增排期
4. 当天打卡完成
5. 查看首页变化
6. 查看统计变化
7. 查看奖励变化
8. 切换另一个账号验证隔离

## 七、缺陷提交建议

建议缺陷按以下维度分类：

- 阻断：无法使用核心链路
- 严重：功能能进但结果错误
- 一般：体验问题、文案问题、局部逻辑问题
- 建议：样式优化、交互优化
