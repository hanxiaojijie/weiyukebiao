## 二、测试执行总表字段说明

为了避免测试人员理解不一致，建议按下面规则填写：

| 字段 | 是否必填 | 填写说明 |
| --- | --- | --- |
| 测试编号 | 是 | 必须与测试用例文档中的编号一致，例如 `TC-A-001` |
| 模块 | 是 | 如登录、首页、课程、课表、打卡、奖励、统计、个人中心 |
| 优先级 | 是 | `P0 / P1 / P2` |
| 测试内容 | 是 | 用一句话概括本条用例要验证什么 |
| 测试方法 | 是 | 建议填写：手工功能测试 / 业务流测试 / 边界测试 / 异常测试 / 隔离测试 / 回归测试 |
| 前置条件 | 是 | 测试开始前必须满足的条件，如账号、数据、页面状态 |
| 测试步骤 | 是 | 用 1、2、3 的形式写清楚操作过程，不能省略关键动作 |
| 预期结果 | 是 | 建议同时写页面结果和关键数据结果 |
| 测试结论 | 是 | 建议统一用：通过 / 不通过 / 阻塞 |
| 缺陷单号 | 否 | 如果失败并已提单，则填写缺陷编号 |
| 执行人 | 建议填写 | 便于追踪是谁执行的 |
| 执行日期 | 建议填写 | 便于追踪版本回归时间 |
| 备注 | 否 | 可填写特殊说明，例如“仅 Android 复现” |

## 三、测试方法填写建议

不同测试内容，建议这样归类：

| 测试方法 | 适用场景 | 示例 |
| --- | --- | --- |
| 手工功能测试 | 验证页面功能是否可用 | 新增课程、保存规则 |
| 业务流测试 | 验证跨页面链路是否闭环 | 打卡后首页、奖励、统计是否同步 |
| 边界测试 | 验证上限、下限、空值、非法值 | 时间反填、目标学分为 0 |
| 异常测试 | 验证错误处理是否合理 | 请假不填原因、休息次数超限 |
| 隔离测试 | 验证不同账号数据隔离 | 账号 A 与账号 B 切换 |
| 回归测试 | 验证修复后主链路未受影响 | 修复 bug 后重走主流程 |

## 四、测试结论填写建议

建议统一使用以下标准：

| 测试结论 | 含义 |
| --- | --- |
| 通过 | 实际结果与预期结果一致 |
| 不通过 | 实际结果与预期结果不一致，且已确认为问题 |
| 阻塞 | 当前环境、账号、数据或前置问题导致本条无法继续执行 |

## 五、用例执行规范

### 优先级定义

- `P0`：主链路必测，不通过不能提测通过
- `P1`：重要功能，建议本轮覆盖
- `P2`：体验或补充验证

### 执行记录建议

测试执行时建议增加以下结果字段：

- 实际结果
- 是否通过
- 缺陷单号
- 备注

## 模块 A：登录与账号初始化

### TC-A-001 首次进入自动初始化

- 优先级：`P0`
- 测试目标：验证新账号首次进入时可以正常初始化
- 前置条件：使用一个从未进入过该小程序的微信账号
- 测试步骤：
  1. 打开小程序体验版
  2. 等待首页加载完成
  3. 进入“我的”页面
- 预期页面结果：
  - 小程序可正常进入，不出现白屏、报错、无限 loading
  - 首页正常展示默认内容
  - “我的”页面可正常打开
- 预期数据结果：
  - 用户个人数据初始化成功
  - 不需要手工注册即可使用

### TC-A-002 老账号再次进入读取原数据

- 优先级：`P0`
- 测试目标：验证已有用户再次进入能读到原有数据
- 前置条件：账号 A 已创建过课程、排期或打卡记录
- 测试步骤：
  1. 使用账号 A 打开小程序
  2. 查看首页
  3. 查看课程页、我的页
- 预期页面结果：
  - 已有课程、排期、规则、奖励内容能正常展示
  - 不会被重置成新用户状态
- 预期数据结果：
  - 原有个人数据完整保留

### TC-A-003 双账号数据隔离

- 优先级：`P0`
- 测试目标：验证不同用户的个人数据不会串
- 前置条件：
  - 账号 A 已创建课程排期并完成至少 1 条打卡
  - 准备账号 B
- 测试步骤：
  1. 用账号 A 查看自己的首页、打卡页、统计页
  2. 退出或切换到账号 B
  3. 进入首页、课程页、打卡页、统计页
- 预期页面结果：
  - 账号 B 看不到账号 A 的个人排期、个人打卡、个人规则、个人奖励
  - 共享课程库仍然可以查看
- 预期数据结果：
  - 数据按用户隔离，不发生串号

## 模块 B：课程录入与课程详情

### TC-B-001 新增课程成功

- 优先级：`P0`
- 测试目标：验证课程可以成功录入
- 前置条件：已登录账号 A
- 测试数据：
  - 课程名称：考研英语
  - 类别：语言学习
  - 总时长：60
  - 描述：考研英语长期复习课程
- 测试步骤：
  1. 进入“课程”页
  2. 点击“录入课程”
  3. 填写课程信息并保存
  4. 返回课程页
- 预期页面结果：
  - 保存成功提示出现
  - 课程卡片出现在共享课程库中
- 预期数据结果：
  - 新课程被成功写入课程库

### TC-B-002 课程必填校验

- 优先级：`P0`
- 测试目标：验证课程录入的必填校验有效
- 前置条件：已进入课程编辑页
- 测试步骤：
  1. 不填写课程名称
  2. 直接点击保存
- 预期页面结果：
  - 页面提示课程信息不完整或类似校验提示
  - 不允许保存成功
- 预期数据结果：
  - 课程库中不新增空课程

### TC-B-003 课程详情展示正确

- 优先级：`P1`
- 测试目标：验证课程详情页字段展示正确
- 前置条件：已有一门完整课程
- 测试步骤：
  1. 在课程页点击某课程
  2. 进入课程详情页
- 预期页面结果：
  - 课程名称、类别、描述、总时长、资料信息展示正确
  - 页面无空白或布局错乱
